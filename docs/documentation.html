Getting Started

Installation

Proteus Cursor works with any framework or plain HTML. No build tools required for the UMD version.

npm
$ npm install proteuscursor
CDN / HTML <script> tag
<link rel="stylesheet" href="proteus-cursor.min.css"> <script src="proteus-cursor.umd.js"></script>
💡
Always include the CSS file — it contains the base styles for both cursor modes. Without it the cursor shape won't be visible.
Getting Started

Quick Start

Three lines of code to get your first shapeshifting cursor.

ES Module
import ProteusCursor from 'proteuscursor'; import 'proteuscursor/style'; const cursor = new ProteusCursor({ shape: 'circle', shape_color: '#00D4AA', click_animation: 'ripple', }); // Or start from a preset const cursor2 = new ProteusCursor({ ...ProteusCursor.getPreset('neon') });
Core Concepts

The Two Modes

All cursor behaviour in Proteus builds on one of two base modes, set via the shape option. Everything else — color, size, shadow, text, presets — is layered on top.

circle mode
new ProteusCursor({ shape: 'circle', // perfectly round dot + lagging shadow shape_size: '12px', shape_color: '#fff', hasShadow: true, shadow_size: '44px', });
fluid mode
new ProteusCursor({ shape: 'fluid', // physics blob, stretches with velocity shape_size: '24px', shape_color: '#e8e8e8', speed: 0.9, // lerp factor 0–1 maxVelocity: 10, });

You can switch modes at runtime with cursor.setShape('fluid') or cursor.setShape('circle').

Core Concepts

State Machine

Register named cursor states in JavaScript with addState(), then activate them anywhere in HTML with a single data-cursor-state attribute. States revert automatically on mouse leave.

JavaScript — register states
const cursor = new ProteusCursor({ shape: 'circle' }); cursor .addState('gallery', { shape_size: '80px', text: 'Zoom', text_size: '13px', text_color: '#fff', }) .addState('video', { shape_size: '90px', text: '▶ Play', hasShadow: false, }) .addState('cta', { shape_size: '70px', shape_color: 'rgba(255,107,53,0.2)', text: 'Click', text_color: '#FF6B35', text_weight: '600', });
HTML — activate with data attribute
<!-- Activates 'gallery' state on hover --> <div class="gallery-item" data-cursor-state="gallery"> <img src="photo.jpg" /> </div> <!-- Activates 'video' state on hover --> <div data-cursor-state="video"> <video src="clip.mp4"/> </div>

addState() is chainable. Remove a state at runtime with cursor.removeState('gallery').

Core Concepts

Data Attributes

Per-element overrides without registering a named state — useful for one-off hover effects.

AttributeDescription
data-cursor-state="name"Activates a registered state on hover. Reverts on mouse leave.
data-proteus-shapeSize="40px"Override shape size on hover for this element only.
data-proteus-shapeColor="#FF0"Override shape color on hover for this element only.
data-proteus-text="Open"Display text inside cursor when hovering this element.
data-proteus-textColor="#fff"Text color for this element's hover state.
data-proteus-textSize="14px"Text font size for this element's hover state.
data-proteus-textWeight="700"Text font weight for this element's hover state.
data-proteus-shadowIsEnabled="false"Disable shadow when hovering this element.
Example — no state registration needed
<button data-proteus-shapeSize="48px" data-proteus-shapeColor="rgba(255,107,53,0.2)" data-proteus-text="Go!" data-proteus-textColor="#FF6B35" > Submit </button>

Usage

Features

Blend Mode

Apply CSS mix-blend-mode to the cursor shape via the blend_mode option. The difference value creates automatic color inversion — the cursor always stays readable against any background.

dark background
light background

With blend_mode: 'difference' and a white cursor, the shape inverts automatically on both halves.

Constructor
new ProteusCursor({ shape: 'circle', shape_size: '40px', shape_color: '#ffffff', hasShadow: false, blend_mode: 'difference', }); // Or at runtime cursor.setBlendMode('difference'); cursor.setBlendMode('normal'); // reset
Common values
ValueDescription
'normal'Default. No blending effect.
'difference'Inverts colors beneath the cursor. Best with a white shape.
'exclusion'Softer inversion, lower contrast than difference.
'multiply'Darkens underneath the cursor.
'screen'Lightens underneath the cursor.
any CSS valueAll mix-blend-mode values are supported.
Features

Click Animations

Three built-in click feedback styles. Set at construction time or change at runtime.

ValueDescription
'scale'The cursor punches inward on click — a satisfying physical press feeling.
'ripple'An expanding ring radiates outward from the cursor position on click.
'none'No click feedback. For pure visual cursors with no interaction emphasis.
Example
new ProteusCursor({ shape: 'circle', click_animation: 'ripple', click_duration: 350, // ms }); // Change at runtime cursor.click_animation = 'scale';
Integrations

Framework Adapters

First-class hooks for React, Vue 3, and Svelte via subpath exports. Each adapter handles the mount/unmount lifecycle automatically and is SSR-safe (Next.js, Nuxt, SvelteKit).

// Install npm install proteuscursor import { useProteusCursor } from 'proteuscursor/react'; import 'proteuscursor/style'; export default function App() { const cursor = useProteusCursor({ shape: 'circle', shape_color: '#ffffff', click_animation: 'ripple', }); // cursor.current → ProteusCursor instance (null until mounted) return <main>...</main>; }
import { useProteusCursor } from 'proteuscursor/vue'; import 'proteuscursor/style'; // inside <script setup> const cursor = useProteusCursor({ shape: 'circle', shape_color: '#ffffff', click_animation: 'ripple', }); // cursor.value → ProteusCursor instance (null until mounted)
import { useProteusCursor } from 'proteuscursor/svelte'; import 'proteuscursor/style'; // inside <script> block (Svelte 4 & 5) const cursor = useProteusCursor({ shape: 'circle', shape_color: '#ffffff', click_animation: 'ripple', }); // cursor.current → ProteusCursor instance (undefined until mounted)

✅ SSR-safe  ·  ✅ TypeScript types included  ·  ✅ Full API accessible via returned ref

📦
Peer dependencies — React, Vue and Svelte are peer dependencies. Install only the one you need. The adapters are tree-shaken from your bundle automatically.
Accessibility

Touch & Reduced Motion

Proteus Cursor is designed to be inclusive by default. Both behaviours are opt-out via constructor options if you need manual control.

📱
Touch Device Support

On touch-only devices (phones, tablets without a paired mouse), the constructor returns immediately without injecting any DOM elements or attaching event listeners. The native cursor — if any — remains unaffected.

♿
Reduced Motion

When the OS has prefers-reduced-motion: reduce set, the constructor also skips init entirely. No animation frames, no overlay elements. Disable with respectReducedMotion: false if your site itself handles this flag.

Manual detection
// Check before initialising in custom logic if (!ProteusCursor.isTouchOnly() && !ProteusCursor.prefersReducedMotion()) { const cursor = new ProteusCursor({ shape: 'circle' }); } // Or opt out of the automatic check const cursor = new ProteusCursor({ shape: 'circle', respectReducedMotion: false, // handle it yourself });
💡
Both static methods are safe to call server-side — they check for typeof window and return false in SSR environments.
